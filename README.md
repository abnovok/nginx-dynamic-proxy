# Динамический Nginx Reverse Proxy Стенд (Защита от IP Spoofing)

Решение тестового задания по организации цепочки проксирования через несколько инстансов Nginx с динамическим определением роли балансировщика (`Ingress` / `Internal`) и защитой от подмены IP-адресов (IP Spoofing) в заголовке `X-Forwarded-For`.

## 🚀 Архитектурное решение

Главная фича стенда — **универсальность конфигурации**. В соответствии с требованием, что запрос может прийти на любой Nginx в цепочке, инстансы не разделяются на жесткие роли на этапе деплоя. 

Каждая нода динамически определяет свой статус на основе IP-адреса отправителя (`$remote_addr`):
1. **Режим Ingress (Граница доверия):** Если запрос пришел с внешнего (недоверенного) IP, Nginx принудительно затирает входящий `X-Forwarded-For`, предотвращая spoofing, и записывает туда реальный IP клиента.
2. **Режим Internal (Доверенная зона):** Если запрос пришел из внутренней сети Docker (от соседнего Nginx), нода доверяет заголовку и дописывает IP предыдущего прокси в цепочку с помощью `$proxy_add_x_forwarded_for`.

Реализовано с помощью модулей `ngx_http_geo_module` и `ngx_http_map_module`.

## 📂 Структура репозитория

* `config/nginx.conf` — единый конфигурационный файл для всех контейнеров Nginx.
* `app/` — минималистичное Python/Flask приложение, которое выводит все полученные HTTP-заголовки для валидации.
* `scripts/test.sh` — автоматизированный Bash-скрипт для проверки работоспособности и безопасности.
