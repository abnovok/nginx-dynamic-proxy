# Динамический Nginx Reverse Proxy Стенд (Защита от IP Spoofing)

Решение тестового задания по организации цепочки проксирования через несколько инстансов Nginx с динамическим определением роли балансировщика (`Ingress` / `Internal`) и защитой от подмены IP-адресов (IP Spoofing) в заголовке `X-Forwarded-For`.

---

## 🛠 Архитектурные особенности решения

### 1. Универсальный динамический конфиг (`DRY` на практике)
В соответствии с требованием, что запрос может прийти на любой Nginx в цепочке, инстансы не разделяются на жесткие роли при деплое. Каждая нода использует **один и тот же** конфигурационный файл и динамически определяет свой статус на основе IP-адреса отправителя (`$remote_addr`):
* **Режим Ingress (Граница доверия):** Если запрос пришел с внешнего (недоверенного) IP, Nginx принудительно затирает входящий `X-Forwarded-For`, предотвращая spoofing, и записывает туда реальный IP клиента.
* **Режим Internal (Доверенная зона):** Если запрос пришел из внутренней сети Docker (от соседнего Nginx), нода доверяет заголовку и дописывает IP предыдущего прокси в цепочку.

Реализовано с помощью модулей `ngx_http_geo_module` и `ngx_http_map_module`.

### 2. Параметризация конфигурации через шаблоны
Чтобы избежать дублирования кода и хардкода адресов, файл конфигурации хранится в виде шаблона `config/nginx.conf.template`. 
При старте контейнеров официальный образ Nginx с помощью утилиты `envsubst` динамически подставляет переменную окружения `${NEXT_HOP}`, указывая каждой ноде путь к следующему шагу в цепочке. При этом «родные» переменные Nginx защищены от перезаписи с помощью фильтра `NGINX_ENVSUBST_FILTER`.

### 3. Полиглот-бэкенд: Python & Go 
В каталоге `app/` подготовлены две модифицированные заготовки бэкенда: на **Python (Flask)** и на **Go (Native HTTP)**.
* Сделал для демонстрации полной независимости инфраструктурного слоя от стека разработки. Архитектуре проксирования всё равно, что находится в конце цепочки — интерпретируемый скрипт или легковесный скомпилированный бинарник. Оба приложения настроены на сквозное логирование HTTP-заголовков в `stdout` в реальном времени.
* Образ Go собран с применением **Multi-stage build**, что минимизирует размер финального контейнера до ~15 МБ и исключает лишние зависимости.
